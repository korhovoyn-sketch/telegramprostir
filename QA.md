# QA / Verification — PropSpace

Реліз-чек-лист для Telegram Mini App. Дві частини: **(1) автоматизовані тести**
(герметичні, мок-бекенд) і **(2) ручні перевірки**, які можливі лише на реальному
пристрої в Telegram із живим Supabase.

> Правило релізу: жоден реліз не виходить, поки всі автотести зелені
> (`npm run test:all`) **і** ручний чек-лист пройдено на iOS **та** Android.

---

## 1. Автоматизовані тести

| Шар | Команда | Що покриває | Де виконується |
|---|---|---|---|
| Типи | `npm run type-check` | TypeScript strict, увесь проект + тести | контейнер / CI |
| Лінт | `npm run lint` | ESLint + next/core-web-vitals | контейнер / CI |
| Збірка | `npm run build` | static export `out/` без помилок | контейнер / CI |
| Unit | `npm run test` | `utils`, навігація `appStore`, логіка `useAuth` | контейнер / CI |
| Component | `npm run test` | Welcome / RoleSelect / ProfileSetup рендер + контракти | контейнер / CI |
| E2E | `npm run test:e2e` | повні флоу в реальному Chromium, **guard видимості кнопок** | лише CI* |

\* Playwright-браузер тягнеться з CDN — у середовищі без мережі E2E не запускається
локально, лише в GitHub Actions (`.github/workflows/ci.yml`).

**Один прогін усього:** `npm run test:all`.

### Що саме ловлять автотести (регресії з історії розробки)

- **«Кнопку входу не видно / сплюснута»** → `tests/e2e/layout-buttons.spec.ts`:
  `boundingBox()` кожної головної кнопки на iPhone SE — height ≥ 40 і в межах
  viewport. Component-рівень: `mbtn` не має `position:relative` override.
- **«Застрягання на спінері Авторизація…»** → `tests/e2e/auth-loading.spec.ts`
  (повільний Edge Function → видно кроковий лоадер) + `tests/unit/useAuth.restore*`
  (Fast Path 0: кеш-профіль відновлює сесію **без** мережевого логіну).
- **`updateProfile` контракт (boolean, не throw)** → `tests/unit/useAuth.updateProfile`
  + `tests/components/RoleSelectScreen` / `ProfileSetupScreen` (навігація лише на `true`).
- **Self-escalation (A04)** → `updateProfile` стрипить `plan`/`id`/`tg_id`.
- **Навігація не повертає на екран логіну** → `tests/unit/appStore` (`back` фільтрує
  `splash`/`welcome`/`role-select`; history капиться на 12).

## 2. Ручні перевірки (лише на пристрої в Telegram)

Те, що **неможливо** перевірити герметично. Прогнати на **iOS і Android**.

### 2.1 Авторизація та сесія
- [ ] Холодний старт без сесії: splash → welcome → авто-логін → дашборд.
- [ ] **iOS:** вбити застосунок повністю, відкрити знову → миттєве відновлення
      сесії з CloudStorage (без повторного логіну, без спінера).
- [ ] Прострочений токен (>7 днів): відкриття → Fast Path 0 відновлює профіль.
- [ ] Реальна HMAC-валідація: логін проходить лише через справжній `initData`.
- [ ] Logout → welcome; повторний вхід працює.

### 2.2 RLS / безпека (Supabase SQL Editor + два акаунти)
- [ ] Owner бачить **лише свої** бази/об'єкти.
- [ ] Realtor бачить **лише підписані** бази.
- [ ] `prevent_privilege_escalation`: спроба realtor→owner — блок; зміна `plan` з
      клієнта — ігнор.
- [ ] Share-token expiry: прострочене посилання → «Посилання недійсне».
- [ ] Storage: фото вантажиться лише в шлях `{propertyId}/…`.

### 2.3 Дизайн / верстка на пристрої (доповнює E2E)
- [ ] iPhone SE (маленький екран): жодна головна кнопка не обрізана, не перекриває контент.
- [ ] Safe-area: FAB і нижні кнопки над home-indicator (не під ним).
- [ ] Клавіатура iOS: активний інпут скролиться у видиму зону, кнопка не перекрита.
- [ ] Темна тема Telegram: кольори/контраст коректні.
- [ ] Telegram BackButton: показ/хов і повернення на правильний екран.

### 2.4 Основні флоу
- [ ] Owner: створити базу → додати об'єкт → поділитись QR → аналітика → експорт PDF.
- [ ] Realtor: сканувати QR → підписка → перегляд бази → підбірка → поділитись.
- [ ] Deep-links: `db_`, `prop_`, `col_` (гостем і залогіненим).
- [ ] Сповіщення: позначити прочитаним → лічильник зникає.
- [ ] Офлайн-банер з'являється/зникає при втраті/відновленні мережі.
